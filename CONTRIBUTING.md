# 🤝 Contribuir al Sistema de Gestión de Producción

¡Gracias por tu interés en contribuir! Este documento te guiará sobre cómo participar en el desarrollo del proyecto.

## 🚀 Cómo Contribuir

### 1. Fork del Repositorio
1. Haz fork del repositorio
2. Clona tu fork localmente
3. Crea una rama para tu funcionalidad

```bash
git checkout -b feature/nueva-funcionalidad
```

### 2. Realizar Cambios
- Sigue las convenciones de código existentes
- Agrega comentarios claros
- Prueba tus cambios localmente

### 3. Commit y Push
```bash
git add .
git commit -m "✨ Agregar nueva funcionalidad"
git push origin feature/nueva-funcionalidad
```

### 4. Crear Pull Request
- Describe claramente los cambios
- Incluye capturas de pantalla si hay cambios visuales
- Referencia issues relacionados

## 📋 Convenciones

### Mensajes de Commit
Usa emojis para categorizar commits:
- ✨ `:sparkles:` Nueva funcionalidad
- 🐛 `:bug:` Corrección de bugs
- 📚 `:books:` Documentación
- 🎨 `:art:` Mejoras de UI/UX
- ♻️ `:recycle:` Refactoring
- 🔧 `:wrench:` Configuración
- 🚀 `:rocket:` Despliegue

### Código PHP
- Seguir PSR-12
- Documentar funciones públicas
- Usar nombres descriptivos

### Código JavaScript
- Usar ES6+
- Comentarios JSDoc para funciones
- Mantener consistencia con el estilo existente

## 🧪 Pruebas

Antes de enviar un PR:
- [ ] Probar funcionalidad nueva
- [ ] Verificar que no se rompa funcionalidad existente
- [ ] Probar en diferentes navegadores
- [ ] Verificar responsive design

## 📞 Contacto

Si tienes preguntas:
- Abre un Issue para discusión
- Revisa Issues existentes
- Consulta la documentación
